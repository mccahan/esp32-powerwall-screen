‚öôÔ∏è Configuration

First Boot Setup:
  1. After flashing, the device will create a WiFi access point named Powerwall-Display
  2. Connect to this access point with password configure
  3. A captive portal will open automatically (or navigate to 192.168.4.1)
  4. Configure your WiFi credentials and MQTT server settings:
    • MQTT Server: Your MQTT broker IP or hostname
    • MQTT Port: Usually 1883
    • MQTT User/Password: Your broker credentials (if required)
    • MQTT Topic Prefix: The prefix for your topics (e.g., "powerwall" or "pypowerwall")
  5. Save and the device will connect to your WiFi and MQTT broker

üì° MQTT Topics

The display subscribes to the following MQTT topics (where {prefix} is your configured prefix):

  • {prefix}/site/instant_power - Grid power in watts (positive = importing, negative = exporting)
  • {prefix}/solar/instant_power - Solar generation in watts
  • {prefix}/battery/instant_power - Battery power in watts (positive = discharging, negative = charging)
  • {prefix}/battery/level - Battery state of charge as percentage (e.g., 50.4)
  • {prefix}/load/instant_power - Home consumption in watts

üé® Display Layout

  • Grid - Left side (White)
  • Solar - Top (Yellow)
  • Battery - Bottom (Green)
  • Home - Right side (Blue)

Energy flow lines animate between components based on real-time power data.

üêõ Troubleshooting

Device not appearing in COM port list?
  • Make sure you're using a data-capable USB cable (not power-only)
  • Install CH340 or CP2102 USB drivers if needed
  • Try a different USB port
Display not working?
  • Verify you have the correct hardware (ESP32-S3 with ST7701S display)
  • Check that backlight is connected and powered
  • Ensure display ribbon cable is properly seated
